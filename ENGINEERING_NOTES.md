# ENGINEERING NOTES

## Macro value
LEB is a governed meta-engine for designing and operating other LLM engines.
Its macro value is that it turns prompt construction into architecture with:
- lifecycle management
- diagnostics
- performance evaluation
- bounded optimization
- role-based expertise
- simulation-backed change control

## Why v11.8 exists
v11.7 compressed too aggressively and removed structural detail. v11.8 fixes
that by using the real v11.6 engine as the preservation baseline and layering
new capabilities on top.

## Engineering posture
- preserve command families
- preserve registries
- preserve governance boundaries
- add features as layered expansions
- bound team mode with strict guardrails
- keep reporting and normalization explainable

## Team mode implementation strategy
- hard cap on agents and rounds
- scope-locked deliberation
- structured contributions
- forced convergence
- lead-agent or operator resolution
